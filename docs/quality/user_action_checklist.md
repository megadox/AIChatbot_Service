# 사용자 실행 체크리스트 (챗봇 품질 전면 개선)

이 문서는 `챗봇_품질_전면_개선` 계획의 **사용자 할 일**만 모아 즉시 실행 가능하도록 정리한 체크리스트입니다.

## 0단계/1단계 결정사항 (완료)

- [x] 우선 개선 도메인 확정: `WIN32`, `WEB`, `EXCEL`, `COMMON`, `BuiltIn`
- [x] 불확실 질의 정책: `정확도 우선(모르면 되묻기)`
- [x] 기본 응답 스타일: `균형형`

---

## 2단계: 검색 정확도 상향 (사용자 입력 필요)

### A. 도메인별 오답 사례 30~50건 제공
- [O] 오답 사례 수집 완료
- [O] 아래 JSON 포맷으로 저장

권장 파일: `qa/user_error_cases.json`

```json
[
  {
    "id": "user-case-001",
    "domain": "WEB",
    "question": "사용자 실제 질문",
    "expectedSource": "WEB/Click.md",
    "actualTopSource": "SAP/Click.md",
    "whyWrong": "브라우저 맥락인데 SAP가 선택됨"
  }
]
```

### B. 플랫폼 의도 우선 규칙 승인
- [O] 승인 여부 선택
  - 권장: 질문에 `윈도우/.exe/cmd/powershell` 있으면 `WIN32` 우선
  - 권장: 질문에 `android/apk/패키지/안드로이드` 있으면 `ANDROID` 우선

---

## 3단계: 임베딩/KB 최적화 (사용자 입력 필요)

### C. 운영 제약 확정
- [O] 허용 KB 재생성 시간(분): `90`
- [O] 허용 최대 메모리(GB): `16`
- [O] 운영 시간대 제약(예: 업무시간 제외): `제약없음`

### D. 모델 교체 가능 여부
- [O] 현 모델 고정
- [X] 모델 교체 가능
  - 후보/라이선스 메모: `_____`

---

## 4단계: 품질 게이트 정책 (사용자 입력 필요)

### E. 게이트 임계값 승인
- [O] Retrieval 신규 회귀 허용(`MaxFail`): `0` (권장 0)
- [O] Answer Eval 허용(`MaxAnswerFail`): `0` (권장 0)
- [O] Fail rate 허용치: `0.0` (권장 0.0)

### F. 릴리즈 블로킹 기준
- [ ] 아래 중 선택
  - [ ] 검색 FAIL만 있어도 배포 차단
  - [ ] 답변 FAIL만 있어도 배포 차단
  - [O] 둘 중 하나라도 FAIL이면 배포 차단(권장)
  - [ ] 경고만 표시, 배포는 허용

---

## 5단계: 운영 교정 루프 (사용자 입력 필요)

### G. 로그 수집/보관 정책
- [O] 수집 항목 승인
  - [O] 질문 텍스트
  - [O] 선택 source
  - [O] 실패 이유
  - [O] 교정(`정답:`) 이벤트
- [O] 개인정보 마스킹 규칙: `없음`
- [O] 보관 기간(일): `30`

### H. 월간 리뷰 리듬
- [O] 리뷰 주기
  - [O] 주 1회(권장)
  - [ ] 격주 1회
  - [ ] 월 1회
- [O] 고정 리뷰 요일/시간: `금요일 오후 12시30분`

---

## 작성 후 전달 방법

1. 이 파일의 빈칸을 채워 저장합니다.
2. 채운 뒤 채팅에 아래처럼 알려주세요:
   - `docs/quality/user_action_checklist.md 작성 완료`
3. 그러면 제가 입력값을 코드/스크립트/운영 규칙에 반영하는 작업을 바로 이어서 진행합니다.
