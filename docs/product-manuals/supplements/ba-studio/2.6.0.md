## Source Code Supplement

이 섹션은 BA-Studio 실제 소스 `E:\BATEM_RPA\BATEM\DXLogicEditor`를 확인하여 온라인 매뉴얼의 누락되기 쉬운 운영 정보를 보강한 내용이다.

### Source-Confirmed Project and Library Creation

- 새 Project 또는 Library 생성 화면은 `UI/NewProject.cs`에서 확인된다.
- 생성 시 사용자는 저장 폴더와 이름을 입력해야 하며, 폴더 또는 이름이 비어 있으면 생성이 중단된다.
- 저장 경로가 존재하지 않거나 잘못된 드라이브이면 생성이 중단된다.
- 동일한 이름의 폴더가 이미 있으면 생성할 수 없다.
- 생성 성공 시 선택한 폴더 아래에 Project 또는 Library 이름의 폴더가 만들어지고, 내부에 `Resource` 폴더가 함께 생성된다.
- Library 생성 모드에서는 화면 제목과 안내 문구가 `Library` 기준으로 바뀐다.
- Server 라이선스 환경에서는 Library 이름 중복을 서버에서 확인하려고 시도하며, 서버 연결 실패 시 사용자 확인 후 계속 생성할 수 있다.

### Source-Confirmed Activity Library Loading

- 기본 액티비티 라이브러리는 `Model/ApplicationContext.cs`에서 `Resources\commands.json`을 기준으로 로드된다.
- 실행 위치에 따라 `BA-Studio\Resources\commands.json` 하위 경로도 fallback으로 사용된다.
- 사용자 정의 액티비티는 custom commands 파일을 별도로 읽어 기본 라이브러리 뒤에 병합된다.
- 액티비티 설정 정보는 `Resources\commands_config.json`에서 로드된다.
- 액티비티 설명/가이드 후보 파일은 `Resources\commands_guide.yaml`로 정의되어 있다.

### Source-Confirmed Project Bundle Structure

- 열린 태스크/프로젝트 컨텍스트는 `Model/Context.cs`에서 Bundle 경로를 기준으로 리소스 경로를 계산한다.
- `resource` 폴더는 일반 리소스의 기준 경로이다.
- `resource\images` 폴더는 이미지 리소스의 기준 경로이다.
- `libs` 폴더는 사용자 정의 라이브러리 파일의 기준 경로이다.
- `libs_exe` 폴더는 코드 실행용 임시 라이브러리 경로로 사용된다.
- `info\images` 폴더는 안내 이미지 경로로 사용된다.
- `Selector` 폴더는 셀렉터 관련 파일 경로로 사용된다.

### Source-Confirmed Run and Debug Behavior

- 실행/디버그 처리는 `Model/Controller.cs`의 `Execute` 흐름에서 확인된다.
- 실행 전 기본적으로 현재 문서를 저장한다.
- 디버그 실행이면 실행 모드가 `DEBUGING`, 일반 실행이면 `EXECUTING`으로 설정된다.
- 디버그 실행은 `RUNNING_MODE.LINE_BY_LINE`, 일반 실행은 `RUNNING_MODE.BATCH`로 시작된다.
- BreakPoint 목록은 실행 컨텍스트에 전달된다.
- Project 전체 실행, Project 안의 단일 Task 실행, Library 모드 Task 실행, 단일 fp 파일 실행이 분기되어 처리된다.
- 특정 액티비티부터 실행하는 경우 `DebugFromHereAsyncTask` 계열 호출이 사용된다.
- 엔진 실행 중 에러 또는 종료 이벤트가 오면 편집 모드로 돌아간다.

### Source-Confirmed Selector and Resource Notes

- 컨텍스트는 `PickedSelector` 값을 별도로 유지하므로, 셀렉터 편집기에서 선택한 셀렉터는 현재 편집 컨텍스트에 반영된다.
- 이미지 리소스는 Bundle 아래 `resource\images`에 저장되는 구조를 전제로 한다.
- 리소스, 이미지, 셀렉터, 사용자 라이브러리는 Project/Task 실행 및 패키징 시 함께 참조되는 보조 파일이다.
