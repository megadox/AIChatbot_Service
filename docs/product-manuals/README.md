# Product Manual Collection Pipeline

This folder is generated by `Tools.ProductManualCollector`.

## Inputs

- `qa/product_manual_sources.json` lists manual pages to collect.
- `--source "Product|Version|https://manual-url"` can be used to collect an ad hoc source without editing the JSON file.

## Command

```powershell
dotnet run --project src\Tools.ProductManualCollector\Tools.ProductManualCollector.csproj
```

Use cached raw HTML and already downloaded images:

```powershell
dotnet run --project src\Tools.ProductManualCollector\Tools.ProductManualCollector.csproj -- --skip-download
```

## Outputs

- `raw/`: downloaded HTML snapshots.
- `assets/`: downloaded manual images.
- `normalized/`: Markdown converted from the manuals.
- `manual_manifest.json`: source URLs, normalized files, image paths, and section paths.
- `reports/manual_audit.json`: image and section audit results.
- `supplements/`: source-code-derived manual additions appended to normalized Markdown.

The next pipeline stage should ingest `normalized/**/*.md` into the chatbot KB as `product_manual` chunks.
